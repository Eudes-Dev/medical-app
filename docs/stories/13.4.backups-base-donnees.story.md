# Story 13.4: Backups et plan de reprise

## Status
Draft

## Story
**As a** praticien,
**I want** que ma base de données soit sauvegardée quotidiennement avec restauration testée,
**so that** je ne perds aucune donnée patient en cas d'incident.

## Acceptance Criteria
1. Backups quotidiens Supabase activés (PITR si plan Pro).
2. Export dump pg vers stockage externe (S3 / Backblaze) chiffré GPG.
3. Procédure de restauration testée trimestriellement (runbook).
4. RPO ≤ 24h, RTO ≤ 4h documenté.

## Tasks / Subtasks
- [ ] Cron `pg_dump` + upload chiffré
- [ ] Doc `docs/ops/disaster-recovery.md`
- [ ] Test restauration en environnement staging
