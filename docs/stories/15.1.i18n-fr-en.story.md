# Story 15.1: Internationalisation FR / EN

## Status
Draft

## Story
**As a** patient anglophone,
**I want** consulter et réserver dans ma langue,
**so that** je comprends l'intégralité du tunnel.

## Acceptance Criteria
1. Toutes les chaînes UI externalisées via `next-intl`.
2. Switch langue dans le header.
3. Locale détectée via `Accept-Language` + persistée en cookie.
4. Formats dates/heures/numéros via `Intl.*`.
5. Emails traduits selon la langue du patient.

## Tasks / Subtasks
- [ ] Setup `next-intl` (App Router)
- [ ] Routes `/[locale]/...`
- [ ] Fichiers `messages/{fr,en}.json`
- [ ] Migration progressive composant par composant
