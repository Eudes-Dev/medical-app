# Story 7.4: Page publique du cabinet

## Status
Draft

## Story
**As a** patient,
**I want** consulter la page d'accueil du cabinet avec coordonnées et services,
**so that** je peux décider de prendre RDV en confiance.

## Acceptance Criteria
1. La route `/` affiche : nom du cabinet, adresse, téléphone, email, photo, horaires d'ouverture, services proposés, CTA "Prendre RDV".
2. Les données proviennent d'une entité `ClinicProfile` éditable depuis `/dashboard/settings/clinic`.
3. La page est référençable (metadata SEO, JSON-LD `MedicalBusiness`).
4. Score Lighthouse SEO ≥ 90 et Performance ≥ 85 sur mobile.

## Tasks / Subtasks
- [ ] Modèle Prisma `ClinicProfile` (singleton)
- [ ] Composant Hero + sections (À propos, Horaires, Services, Localisation Maps)
- [ ] Metadata dynamique via `generateMetadata`
- [ ] JSON-LD structured data
- [ ] Upload photo cabinet (Supabase Storage)
