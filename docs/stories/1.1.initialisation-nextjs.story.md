## Dev Notes

### Stack Technique
- **Framework**: Next.js 16.x avec App Router
- **Language**: TypeScript (mode strict)
- **Styling**: TailwindCSS + Shadcn UI
- **Base de données**: PostgreSQL via Supabase (configuré dans story 1.2)
- **ORM**: Prisma (configuré dans story 1.2)
[Source: docs/project-brief.md]

### Architecture Single-Tenant
Cette application est conçue pour un **cabinet médical unique** (single-tenant). Il n'y a pas de système multi-tenant.

L'application est découpée en deux environnements:
- **Espace Public**: Route `/book/*` pour le tunnel de réservation patient (sans authentification)
- **Espace Privé**: Routes `/dashboard/*` pour le praticien (authentifié)
[Source: docs/front-end-spec/1-architecture-de-linformation-sitemap.md]

### Design System
- Utilisation de TailwindCSS et Shadcn UI
- Palette de couleurs:
  - Principal: Blue-600 (#2563eb) - Actions primaires et santé
  - Neutre: Slate-900 (#0f172a) - Typographie et headers
  - Succès: Emerald-500 - Rendez-vous confirmés
  - Alerte: Amber-500 - En attente ou modification
  - Erreur: Rose-500 - Annulations et erreurs de formulaire
[Source: docs/front-end-spec/2-design-system-ui-components.md]

### Testing
- Tests manuels recommandés pour cette story: vérification du build et du démarrage
- Tests automatisés seront définis dans les stories suivantes
