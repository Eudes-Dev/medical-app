# Story 18.1: Extension multi-praticien (post-MVP)

## Status
Draft (Future)

## Story
**As a** cabinet pluridisciplinaire,
**I want** gérer plusieurs praticiens dans la même instance,
**so that** je n'ai pas à payer plusieurs abonnements.

## Acceptance Criteria
1. Modèle Prisma `User` étendu avec rôles (PRACTITIONER, ASSISTANT, ADMIN).
2. Sélecteur de praticien dans `/book` et le calendrier.
3. Isolation des agendas, partage de la base patients.
4. Permissions granulaires sur les notes de consultation (privé par praticien).
5. Migration depuis le mode single-tenant sans perte de données.

## Tasks / Subtasks
- [ ] Refactor schémas (`practitionerId` sur Appointment, WorkingHours, ServiceType)
- [ ] RBAC via `casl` ou `oso`
- [ ] UI multi-agenda (overlay couleurs)
- [ ] Plan de migration data

## Dev Notes
Story déclassée hors MVP : la vision actuelle est explicitement single-tenant. À reprioriser si demande commerciale.
