# Story 8.4: Rendez-vous récurrents

## Status
Draft

## Story
**As a** praticien,
**I want** créer une série de RDV récurrents pour un patient (suivi hebdomadaire, kinésithérapie, etc.),
**so that** je n'ai pas à les saisir un à un.

## Acceptance Criteria
1. Le formulaire de création propose une option "Répéter" (quotidien, hebdo, mensuel, custom RRULE).
2. Limite : fin après N occurrences OU jusqu'à une date donnée.
3. Modifier une occurrence permet de choisir "cette occurrence" ou "toutes les suivantes".
4. La suppression d'une série supprime ou conserve les occurrences passées (choix).
5. Affichage d'un badge "Série" sur les cartes de RDV.

## Tasks / Subtasks
- [ ] Modèle Prisma `RecurrenceRule` (rrule string, anchorAppointmentId)
- [ ] Génération en série lors du create (limite max 52 occurrences/coup)
- [ ] Logique split "this and following" lors de l'édition
- [ ] UI Shadcn (RecurrenceModal)
- [ ] Tests unitaires des règles
