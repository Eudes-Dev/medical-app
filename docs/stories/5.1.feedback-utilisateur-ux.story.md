# Story 5.1: Feedback Utilisateur (UX)

## Status
Ready for Review

## Story
**As a** utilisateur (praticien, secrétaire ou patient invité),
**I want** avoir des retours visuels clairs pendant les chargements et après chacune de mes actions,
**so that** je sache toujours ce qui se passe, que l'application paraisse réactive et que je puisse corriger mes erreurs sans frustration.

## Contexte métier
La story 2.2 a déjà câblé `sonner` (cf. `components/ui/sonner.tsx` + `<Toaster />` dans `app/layout.tsx`) et la story 2.1 a déjà introduit `components/patients/patient-table-skeleton.tsx`. Cette story consolide l'UX en **généralisant** ces patterns à toute l'application (agenda, tunnel invité, formulaires) et en comblant les manques (état vide, état d'erreur, transitions de page, bouton chargement) avant la livraison MVP (sprint 5 — qualité & déploiement).

## Acceptance Criteria

### Skeletons & chargement
1. Un `CalendarSkeleton` s'affiche pendant le chargement de l'agenda (`app/dashboard/calendar/page.tsx`) via `<Suspense fallback={...}>` ; il reproduit la grille (header jours + 14 lignes de créneaux) avec `animate-pulse`.
2. Le `PatientTableSkeleton` existant (`components/patients/patient-table-skeleton.tsx`) est branché via `<Suspense fallback={...}>` autour du wrapper de table dans `app/dashboard/patients/page.tsx` (au lieu d'un éventuel chargement bloquant) — 10 lignes affichées, identique à la pagination.
3. Tout bouton qui déclenche une mutation (Server Action ou fetch) affiche : un spinner `<Loader2 className="animate-spin" />`, un texte de chargement (prop `loadingText`, défaut « Chargement… »), et passe en `disabled` + `aria-busy="true"`. Implémenté via un composant unique `components/ui/loading-button.tsx` (extension du `Button` Shadcn).
4. Une `loading.tsx` est créée pour chaque segment de route principal afin d'éviter les flashes blancs lors des navigations :
   - `app/dashboard/loading.tsx`
   - `app/dashboard/calendar/loading.tsx`
   - `app/dashboard/patients/loading.tsx`
   - `app/dashboard/patients/[id]/loading.tsx`
   - `app/(public)/[cabinet-slug]/loading.tsx`
   - `app/(public)/[cabinet-slug]/book/loading.tsx`

### Toasts (succès / erreur / info)
5. Un toast `success` (Emerald-500) s'affiche après chacune des actions suivantes — message FR explicite :
   - Création / modification / suppression patient (`app/dashboard/patients/actions.ts`)
   - Création / modification / annulation RDV (`app/dashboard/calendar/actions.ts`)
   - Réservation invité réussie (déjà câblé en 4.2 — vérifier la cohérence du message via le helper centralisé)
6. Un toast `error` (Rose-500) s'affiche en cas d'échec avec un message orienté utilisateur (jamais de stack trace ni d'identifiant interne) :
   - Erreur de validation Zod ⇒ « Vérifiez les informations saisies. »
   - Conflit de créneau (`SLOT_TAKEN`) ⇒ « Ce créneau vient d'être réservé. Merci d'en choisir un autre. »
   - Erreur serveur (`SERVER`) ⇒ « Une erreur est survenue. Merci de réessayer. »
   - Non autorisé (`UNAUTHORIZED`) ⇒ « Votre session a expiré. Reconnectez-vous. »
7. Les helpers de toast sont centralisés dans `lib/ui/toast.ts` (`showSuccess`, `showError`, `showInfo`, `showWarning`) afin d'éviter la duplication de chaînes et de garantir un mapping `error code → message` unique. Les composants n'importent **plus** directement `toast` de `sonner` (sauf le wrapper) — règle vérifiée par un lint manuel listé en DoD.

### États vides
8. Un composant générique `components/ui/empty-state.tsx` est créé avec les props : `icon: LucideIcon`, `title: string`, `description?: string`, `action?: ReactNode`. Il est intégré dans :
   - `components/patients/patient-data-table.tsx` quand la liste est vide ⇒ icône `Users`, « Aucun patient pour le moment », bouton « Nouveau patient ».
   - `components/patients/patient-data-table.tsx` quand la recherche ne retourne rien ⇒ icône `Search`, « Aucun résultat pour « {query} » », bouton « Réinitialiser la recherche ».
   - `components/calendar/CalendarGrid.tsx` (ou conteneur parent) quand aucun RDV sur la journée affichée ⇒ icône `CalendarOff`, « Aucun rendez-vous ce jour », bouton « Nouveau rendez-vous ».
   - `app/(public)/[cabinet-slug]/book/page.tsx` quand `availableSlots = []` ⇒ icône `CalendarX`, « Aucun créneau disponible cette semaine », bouton « Semaine suivante ».

### États d'erreur (page-level)
9. Une `error.tsx` est créée pour chaque segment ci-dessus (Client Component, prop `reset`) affichant un `EmptyState` (icône `AlertTriangle`, message générique, bouton « Réessayer » qui appelle `reset()`). Aucune information technique exposée à l'utilisateur ; l'erreur est loggée via `console.error` (TODO Sentry → story 13.3).
10. Une `not-found.tsx` est créée pour `/dashboard/patients/[id]` et `/(public)/[cabinet-slug]` affichant un `EmptyState` adapté.

### Accessibilité, responsive, rôles
11. Tous les skeletons portent `role="status"` + `aria-live="polite"` + `aria-label="Chargement en cours"` (texte invisible via `sr-only`). Les `<Toaster />` `sonner` exposent par défaut `aria-live="polite"` (succès / info) et `aria-live="assertive"` (erreur) — config vérifiée dans `components/ui/sonner.tsx`.
12. Le `LoadingButton` reste activable au clavier tant qu'il n'est pas en chargement ; en chargement, il est `disabled` mais conserve un focus visible (ring Tailwind) pour éviter la perte de contexte.
13. Tous les composants nouveaux passent en mobile (375px) et desktop (1280px) : `EmptyState` centré avec padding tactile, `CalendarSkeleton` adopte la grille responsive existante.
14. Les patterns sont identiques pour les rôles **praticien**, **secrétaire** et **patient (invité)** — aucune divergence visuelle ni textuelle hors permissions déjà gérées en amont.

### RGPD / sécurité
15. Aucun toast ni `EmptyState` n'affiche de PII (nom, email, téléphone du patient) en dehors de contextes déjà autorisés (ex. confirmation de réservation `success`). Les messages d'erreur sont **génériques côté client** ; le détail (id, payload) reste côté serveur (`console.error`).
16. Les toasts d'erreur n'exposent jamais le détail Prisma / Postgres (codes `P200x`, contraintes uniques, etc.) — le helper `lib/ui/toast.ts` mappe l'erreur en message neutre.

### Arbitrage composants UI — animate-ui PRIORITAIRE
Règle projet **non négociable** : tout composant UI doit venir d'animate-ui (https://animate-ui.com/) en priorité absolue ; Shadcn n'est utilisé qu'en fallback quand un équivalent n'existe pas chez animate-ui ; custom seulement en dernier recours documenté.

| Besoin de la story | Composant retenu | Source | Justification |
|---|---|---|---|
| Skeleton (CalendarSkeleton, PatientTableSkeleton) | `Skeleton` animate-ui (shimmer animé) | animate-ui | Existe, animation intégrée supérieure au `animate-pulse` Shadcn |
| Spinner LoadingButton | `Spinner` (ou `Loader`) animate-ui | animate-ui | Existe ; remplace `Loader2` de lucide |
| Bouton interactif (LoadingButton) | `RippleButton` ou `Button` animate-ui | animate-ui | Existe ; feedback tactile animé |
| Icônes EmptyState (Users, Search, CalendarOff, AlertTriangle) | `lucide-react` | lucide | animate-ui n'expose pas de set d'icônes ⇒ lucide reste autorisé (déjà dans le stack) |
| Conteneur EmptyState | Composant custom `components/ui/empty-state.tsx` | custom (justifié) | animate-ui ne propose pas de pattern « EmptyState » dédié — wrapper minimal autour d'icône + texte + slot action |
| Toast | `sonner` (déjà câblé en 2.2) | Shadcn / sonner | animate-ui ne propose pas de système de toasts ; sonner reste la référence du projet |
| Dialog / Sheet (déjà utilisés en amont) | Shadcn (existant) | Shadcn | Non modifié par cette story — migration éventuelle hors périmètre |

### Architecture — Server Components + Suspense
Tous les `page.tsx` du dashboard sont des Server Components qui appellent des Server Actions async. La stratégie Suspense (au lieu d'un `useEffect + isLoading`) est privilégiée pour bénéficier du streaming Next.js 16 et éviter le double rendu. Modèle :

```tsx
// app/dashboard/calendar/page.tsx (illustratif)
import { Suspense } from "react";
import { CalendarSkeleton } from "@/components/calendar/CalendarSkeleton";
import { CalendarContainer } from "@/components/calendar/CalendarContainer";

export default function CalendarPage() {
  return (
    <Suspense fallback={<CalendarSkeleton />}>
      <CalendarContainer />
    </Suspense>
  );
}
```

```typescript
// lib/ui/error-mapper.ts
import { TOAST_MESSAGES } from "./toast-messages";

type ServerError = { error?: "VALIDATION" | "SLOT_TAKEN" | "SERVER" | "UNAUTHORIZED" };

export function mapServerErrorToMessage(result: ServerError): string {
  switch (result.error) {
    case "VALIDATION":   return TOAST_MESSAGES.errors.validation;
    case "SLOT_TAKEN":   return TOAST_MESSAGES.errors.slotTaken;
    case "UNAUTHORIZED": return TOAST_MESSAGES.errors.unauthorized;
    case "SERVER":
    default:             return TOAST_MESSAGES.errors.server;
  }
}
```

### `LoadingButton` (illustratif — animate-ui)
```tsx
// components/ui/loading-button.tsx
"use client";
// ⚠️ Imports illustratifs — adapter aux exports réels d'animate-ui après installation
import { Button, type ButtonProps } from "@/components/animate-ui/button";
import { Spinner } from "@/components/animate-ui/spinner";

export interface LoadingButtonProps extends ButtonProps {
  isLoading?: boolean;
  loadingText?: string;
}

export function LoadingButton({
  isLoading = false,
  loadingText = "Chargement…",
  children,
  disabled,
  ...rest
}: LoadingButtonProps) {
  return (
    <Button disabled={isLoading || disabled} aria-busy={isLoading} {...rest}>
      {isLoading ? (
        <>
          <Spinner className="mr-2 h-4 w-4" aria-hidden="true" />
          {loadingText}
        </>
      ) : (
        children
      )}
    </Button>
  );
}
```

### `EmptyState` (illustratif)
```tsx
// components/ui/empty-state.tsx
import type { LucideIcon } from "lucide-react";
import type { ReactNode } from "react";

export interface EmptyStateProps {
  icon: LucideIcon;
  title: string;
  description?: string;
  action?: ReactNode;
}

export function EmptyState({ icon: Icon, title, description, action }: EmptyStateProps) {
  return (
    <div className="flex flex-col items-center justify-center text-center py-12 px-4">
      <Icon className="h-12 w-12 text-slate-400" aria-hidden="true" />
      <h3 className="mt-4 text-lg font-medium text-slate-900">{title}</h3>
      {description && <p className="mt-1 text-sm text-slate-500">{description}</p>}
      {action && <div className="mt-6">{action}</div>}
    </div>
  );
}
```
