# Story 9.1: Notes de consultation

## Status
Draft

## Story
**As a** praticien,
**I want** saisir des notes structurées pour chaque consultation,
**so that** je conserve un historique médical fiable.

## Acceptance Criteria
1. Depuis la fiche patient ou la carte de RDV, une zone "Notes de consultation" permet la saisie en Markdown.
2. Les notes sont versionnées (historique des modifications horodaté).
3. Une note est liée à un `Appointment` spécifique.
4. Recherche full-text dans les notes d'un patient.
5. Seul le praticien authentifié peut lire/écrire (jamais exposé côté patient).

## Tasks / Subtasks
- [ ] Modèle Prisma `ConsultationNote` (appointmentId, content, version, createdAt)
- [ ] Éditeur Markdown (Tiptap ou `react-markdown` + textarea)
- [ ] Server Actions CRUD avec versioning
- [ ] Index PostgreSQL `tsvector` pour la recherche
- [ ] Tests sécurité (authz)
