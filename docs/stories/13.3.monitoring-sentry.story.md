# Story 13.3: Monitoring et observabilité (Sentry + logs)

## Status
Draft

## Story
**As a** développeur,
**I want** être alerté en temps réel des erreurs en production,
**so that** je corrige les bugs avant que les utilisateurs ne s'en plaignent.

## Acceptance Criteria
1. Sentry intégré côté client + serveur avec source maps.
2. PII scrubbing actif (pas d'envoi des données patient à Sentry).
3. Logs structurés JSON (pino) avec corrélation par requestId.
4. Tableau de bord Vercel Analytics activé.
5. Alertes Slack/email sur erreurs P1 (taux > X / min).

- [ ] `@sentry/nextjs` setup
- [ ] `beforeSend` hook anti-PII
- [ ] Logger pino + middleware requestId
