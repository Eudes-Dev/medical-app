# Story 7.1: Configuration des horaires d'ouverture

## Status
Draft

## Story
**As a** praticien,
**I want** définir mes horaires d'ouverture par jour de la semaine,
**so that** les patients ne peuvent réserver que sur mes plages travaillées.

## Acceptance Criteria
1. Une page `/dashboard/settings/schedule` permet de définir, pour chaque jour de la semaine, une ou plusieurs plages horaires.
2. Les modifications sont persistées et appliquées immédiatement au tunnel `/book`.
3. La durée par défaut d'un créneau est configurable (15 / 30 / 45 / 60 min).
4. Une plage peut être désactivée sans la supprimer.
5. Validation : les plages d'un même jour ne se chevauchent pas.

## Tasks / Subtasks
- [ ] Modèle Prisma `WorkingHours` (dayOfWeek, startTime "HH:mm", endTime, slotDuration, active)
- [ ] Server Actions CRUD `app/dashboard/settings/schedule/actions.ts`
- [ ] UI avec Shadcn (Tabs par jour + liste de plages)
- [ ] Validation Zod (overlap detection)
- [ ] Refactor générateur de créneaux du tunnel `/book` pour lire `WorkingHours`
- [ ] Tests unitaires (overlap, génération de créneaux)
